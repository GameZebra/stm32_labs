# timerTriggeredADC with Interrupt

A high-performance STM32F407VGT6 project designed to demonstrate precise hardware-triggered Analog-to-Digital Conversion (ADC) with minimal CPU intervention, using interrupts and hardware timers.

---

## 🚀 Project Overview

This project configures **Timer 8 (TIM8)** to generate periodic update events that act as the hardware trigger for **ADC1**. The ADC samples an analog voltage from a potentiometer connected to pin **PA5**. Upon completion of the conversion, an **ADC Interrupt** is fired to process the data. 

Additionally, **Timer 1 (TIM1) Channel 4** is configured as an output on pin **PE14** to provide a visual signal (e.g., PWM or toggle) that can be easily monitored and verified using an oscilloscope.

## 🛠️ Hardware Configurations

| Peripheral | Pin | Function / Connection |
| :--- | :--- | :--- |
| **ADC1** | `PA5` | Analog Input (Connected to Potentiometer) |
| **TIM8** | N/A | Internal Update Event (TRGO) used as ADC trigger |
| **TIM1_CH4** | `PE14` | Timer Output (Connected to Oscilloscope) |

### Clock Configuration
* **System Clock (SYSCLK):** 168 MHz (Maximum allowed frequency for STM32F407)
* **Clock Source:** Configured using the Phase-Locked Loop (**PLL**) driven by the external oscillator (HSE) or internal oscillator (HSI) according to the board setup.

---

## ⚙️ Peripherals Setup Details

### 1. Timer 8 (TIM8) - The Trigger
* Configured in **Master Mode**.
* **Trigger Output (TRGO) Parameters:** Set to *Update Event*.
* The timer is set to auto reload value 10, which gives us 10% resolution (0.3V), and the unneceserily small timer period of 65 nS.
* You can make the timer follow the ADC better by changing the auto reload value to 4095 and removing the conversion in the ADC callback function.

### 2. ADC1 - The Sampler
* **External Trigger Conversion Source:** Configured to *Timer 8 Trigger Out event*.
* **External Trigger Conversion Edge:** Configured to *Trigger detection on the rising edge*.
* **Interrupts:** Enable `ADC global interrupt` in the NVIC settings.

### 3. Timer 1 (TIM1_CH4) - The Visualizer
* Channel 4 is active on pin `PE14`.
* Used to output a signal to verify timing, frequency, or duty cycle changes proportional to the ADC readings on an oscilloscope.

## 💻 How It Works (Code Logic)

1. **Initialization:** The system clocks are boosted to 168 MHz via the PLL. All GPIOs, TIM1, TIM8, and ADC1 are initialized using the generated HAL initialization functions.
2. **Starting Peripherals:** * TIM1 Channel 4 is started to generate the oscilloscope signal.
   * ADC is started in Interrupt mode using `HAL_ADC_Start_IT()`.
   * TIM8 is started to begin generating the TRGO update events.
3. **The Loop:** TIM8 periodically triggers ADC1 automatically in hardware.
4. **Interrupt Handling:** Once the ADC finishes sampling pin `PA5`, the `HAL_ADC_ConvCpltCallback()` function is called. Inside this ISR (Interrupt Service Routine), you can read the converted value and update the TIM1 registers if you wish to modulate the output signal based on the potentiometer.

## 🔍 Verification Setup

To test and verify this project:
1. Connect the center pin of a **potentiometer** to `PA5`, with the side pins connected to `3.3V` and `GND`.
2. Connect your **oscilloscope probe** to pin `PE14` and the ground clip to the board's `GND`.
3. Spin the potentiometer to observe changes or verify the rock-solid timing generated by the hardware-coupled timers.
